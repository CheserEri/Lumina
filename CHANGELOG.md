# 更新日志

## [0.3.0] - 2026-04-05

### 用户认证系统
- 新增用户注册接口 (`POST /api/auth/register`)，支持用户名、邮箱、密码注册
- 新增用户登录接口 (`POST /api/auth/login`)，返回 JWT Token
- 后端集成 `jsonwebtoken` + `bcrypt` 实现安全的密码哈希和 Token 认证
- 用户数据内存存储（`HashMap`），支持多用户隔离

### 前端认证页面
- 新增登录页面 (`LoginScreen`)：简洁居中卡片式设计，支持密码显示/隐藏
- 新增注册页面 (`RegisterScreen`)：包含用户名、邮箱、密码、确认密码表单验证
- 登录/注册页面之间可自由切换
- 侧边栏新增 Sign Out 按钮，退出后返回登录页

### 前端界面重构 (参考 Claudelike 设计)
- 新增左侧固定侧边栏：Logo + New Chat 按钮 + 导航项 + 用户信息
- 顶部导航栏：模型选择器 + 分享/更多操作按钮
- 欢迎页重新设计："What shall we curate today?" 大标题 + 3 张建议卡片（带图标和描述）
- 输入区域采用浮动卡片样式，带阴影和渐变背景过渡
- 底部免责声明采用小写字母 + 宽字间距设计
- 整体配色采用 Claudelike 的优雅灰调色系（`#2d3433`、`#5b5f65`、`#f9f9f8` 等）
- 消息区域全宽交替背景色，每条消息带角色图标和名称标签
- 修复 `ListView` 滚动条问题：使用 `Scrollbar` 包裹，`thumbVisibility: true` 始终显示
- 移除 `itemExtent` 固定高度，消息高度自适应内容

### 架构调整
- 后端统一端口托管前后端，前端通过相对路径请求 API
- 后端新增静态文件服务 (`tower-http::ServeDir`)，通过 `fallback_service` 托管 Flutter Web 构建产物
- 前端 API 服务改用 `dart:html.HttpRequest`（浏览器原生 XHR），支持同源相对路径请求
- 前后端合并为单一端口（默认 3000），跨设备只需开放一个端口

### 网络配置
- Ollama 服务地址改为占位符（`192.168.x.x:11434`），使用时需替换为实际地址
- 服务器监听地址改为 `0.0.0.0:3000`
- 聊天历史保存目录改为占位符（`/tmp/lumina_chats`）
- 所有敏感 IP 已模糊处理，替换为占位符 + 注释说明

### 安全改进
- 仅开放单一端口（3000），API 不直接暴露到外部网络
- 前端通过服务器内部转发与后端通信，Ollama 服务完全隐藏
- 密码使用 bcrypt 哈希存储，Token 使用 JWT 签名验证

### 依赖更新
- 后端新增 `jsonwebtoken`、`bcrypt`、`uuid` 依赖

## [0.2.0] - 2026-04-05

### 架构调整
- 后端统一端口托管前后端，前端通过相对路径请求 API，不再暴露后端端口到外部网络
- 后端新增静态文件服务 (`tower-http::ServeDir`)，通过 `fallback_service` 托管 Flutter Web 构建产物
- 前端 API 服务改用 `dart:html.HttpRequest`（浏览器原生 XHR），支持同源相对路径请求
- 前后端合并为单一端口（默认 3000），跨设备只需开放一个端口

### 前端界面重构 (参考 Gemini/ChatGPT 设计)
- 全新的欢迎页：居中布局 + 渐变 Logo + 大标题 + 建议卡片（带图标）
- 消息区域采用全宽交替背景色（斑马条纹），类似 ChatGPT
- 每条消息左侧显示角色头像 + 名称标签
- 输入框改为圆角胶囊样式，发送按钮使用蓝绿渐变 + 阴影效果
- 底部添加免责声明文字
- 修复 `ListView` 滚动条问题：使用 `Scrollbar` 包裹，`thumbVisibility: true` 始终显示
- 移除 `itemExtent` 固定高度，消息高度自适应内容
- 主题系统优化：Material 3 风格，亮色/暗色主题统一配色

### 网络配置
- Ollama 服务地址改为局域网远程地址（`192.168.x.x:11434`）
- 服务器监听地址改为 `0.0.0.0:3000`
- 聊天历史保存目录改为 Linux 路径（`/path/to/chat/history`）
- 所有敏感 IP 已模糊处理，替换为占位符 + 注释说明

### 安全改进
- 仅开放单一端口（3000），API 不直接暴露到外部网络
- 前端通过服务器内部转发与后端通信，Ollama 服务完全隐藏
