# Lumina计划

## 一、项目概述

本项目旨在构建一个完全本地化运行的高性能 AI 工作站，结合现代系统架构与跨平台 UI 技术，实现一个支持多端访问、流式响应以及专业数学渲染能力的智能对话系统。

系统核心目标包括：

- 本地大模型推理（保障隐私与低延迟）
- 高性能后端服务（高并发 + 内存安全）
- 多端统一 UI（Web + Android）
- 完整支持 Markdown 与 LaTeX 数学公式
- 流式响应优化（类 ChatGPT 体验）

---

## 二、总体架构设计

系统采用分层架构：

```
Flutter (Web / Android)
        ↓
     Rust API 网关
        ↓
     Ollama 本地模型服务
        ↓
 GPU + CPU 硬件加速
```

### 架构说明

- **Flutter 前端**：负责 UI 渲染与交互逻辑
- **Rust 后端**：负责 API 聚合、流式处理与性能优化
- **Ollama**：负责本地大模型推理
- **硬件层**：提供 GPU/CPU 加速能力

---

## 三、技术选型

### 1. 后端技术栈

| 模块 | 技术 |
|------|------|
| Web 框架 | Axum |
| 异步运行时 | Tokio |
| HTTP 客户端 | Reqwest |
| 流式协议 | SSE (Server-Sent Events) |
| 系统监控 | sysinfo |

### 2. 前端技术栈

| 模块 | 技术 |
|------|------|
| UI 框架 | Flutter |
| 状态管理 | Riverpod |
| Markdown 渲染 | flutter_markdown |
| LaTeX 渲染 | flutter_math_fork |

---

## 四、核心功能设计

### 1. 模型推理与 API 转发

Rust 后端作为统一入口：

- 转发请求至 Ollama
- 屏蔽底层实现差异
- 支持未来模型扩展

接口设计：

```
POST /api/chat
GET  /api/models
GET  /api/stream
```

---

### 2. 流式响应（SSE）

系统使用 SSE 实现实时输出：

- Ollama 输出逐 token 返回
- Rust 转换为标准 SSE 格式
- 前端实时渲染

优势：

- 低延迟
- 用户体验接近实时对话

---

### 3. 多端支持

#### Web 端

- 响应式布局
- 支持宽屏（侧边栏 + 聊天区）
- 处理 CORS 问题（通过后端代理）

#### Android 端

- 局域网连接 PC
- 移动端适配（键盘、手势）
- Drawer 侧边菜单

---

### 4. Markdown + LaTeX 渲染

系统支持：

- Markdown 富文本
- 代码高亮
- 数学公式（行内 + 块级）

#### 关键优化：流式渲染问题

问题：
- LaTeX 在未完整输出时会导致渲染错误

解决方案：

- 分段解析文本
- 仅在公式闭合后进行渲染
- 非完整内容使用普通文本展示

---

### 5. 本地数据存储

#### 移动端

- SQLite（sqflite）

#### Web 端

- IndexedDB

数据结构：

```json
{
  "id": "session_id",
  "messages": [
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."}
  ]
}
```

---

### 6. 模型管理

功能：

- 获取本地模型列表
- 动态切换模型

实现：

- 调用 Ollama `/api/tags`
- 前端提供下拉选择

---

### 7. 硬件监控（进阶功能）

监控内容：

- CPU 使用率
- 内存占用
- GPU 使用情况

实现方式：

- CPU：sysinfo
- GPU：调用 nvidia-smi

---

## 五、性能优化策略

### 1. 后端优化

- 全异步架构（Tokio）
- 流式传输减少等待时间
- 并发请求隔离

### 2. 前端优化

- 渲染节流（Debounce）
- 分段更新 UI
- 避免频繁重建组件

### 3. 模型选择建议

| 类型 | 推荐模型 |
|------|----------|
| 轻量 | 7B 模型 |
| 中等 | 8B 模型 |
| 高性能 | MoE 模型 |
