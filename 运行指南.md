# OpenCodeLumina 运行指南

## 项目简介

OpenCodeLumina 是一个基于 Ollama 的智能助手应用，包含前端（Flutter）和后端（Rust）两部分。支持 Markdown 和 LaTeX 数学公式渲染。

---

## 环境要求

### 后端环境
- **Rust**: 1.70 或更高版本
- **Ollama**: 最新版本（需运行在本地）
- **操作系统**: Windows / Linux / macOS

### 前端环境
- **Flutter**: 3.10 或更高版本
- **Dart**: 3.0 或更高版本
- **操作系统**: Windows / Linux / macOS / Android / iOS

---

## 安装步骤

### 1. 安装 Ollama

#### Windows
```bash
winget install Ollama.Ollama
```

#### Linux
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

#### macOS
```bash
brew install ollama
```

### 2. 启动 Ollama 服务

```bash
ollama serve
```

### 3. 下载模型（可选）

```bash
ollama pull qwen3.5:0.8b
```

### 4. 克隆项目

```bash
git clone <repository-url>
cd OpenCodeLumina
```

## 后端运行指南

### 1. 进入后端目录

```bash
cd backend
```

### 2. 编译项目

```bash
cargo build
```

### 3. 运行后端服务

```bash
cargo run
```

或者直接运行编译后的可执行文件：

#### Windows
```bash
.\target\debug\lumina_server.exe
```

#### Linux / macOS
```bash
./target/debug/lumina_server
```

### 4. 验证后端运行

后端启动成功后，会显示以下日志：

```
2026-03-25T16:48:25.009517Z  INFO lumina_server: Starting Lumina Server...
2026-03-25T16:48:25.009570Z  INFO lumina_server: Ollama endpoint: http://localhost:11434
2026-03-25T16:48:25.010380Z  INFO lumina_server: Server listening on 0.0.0.0:8080
```

### 5. 测试后端 API

使用 PowerShell 测试：

```powershell
# 测试获取模型列表
Invoke-WebRequest -UseBasicParsing -Uri http://localhost:8080/api/models | Select-Object -ExpandProperty Content

# 测试非流式聊天
$body = '{"model":"qwen3.5:0.8b","messages":[{"role":"user","content":"Hi"}],"stream":false}'
$headers = @{"Content-Type"="application/json"}
$result = Invoke-RestMethod -UseBasicParsing -Uri "http://localhost:8080/api/chat" -Method Post -Body $body -Headers $headers
Write-Output ($result | ConvertTo-Json -Depth 10)
```

## 配置说明

### 后端配置

后端配置位于 `backend/src/config/mod.rs`，默认配置如下：

```rust
impl Default for AppConfig {
    fn default() -> Self {
        Self {
            ollama: OllamaConfig {
                base_url: "http://localhost:11434".to_string(),
                timeout_secs: 300,
            },
            server: ServerConfig {
                host: "0.0.0.0".to_string(),
                port: 8080,
            },
        }
    }
}
```

**配置项说明**：
- `ollama.base_url`: Ollama 服务地址
- `ollama.timeout_secs`: 请求超时时间（秒）
- `server.host`: 后端服务监听地址
- `server.port`: 后端服务监听端口

### 前端配置

前端 API 服务地址位于 `frontend/flutter_app/lib/services/api_service.dart`：

```dart
class ApiService {
  static const String baseUrl = 'http://localhost:8080';
  // ...
}
```

## 常见问题

### 问题 1：后端无法启动

**错误信息**：
```
通常每个套接字地址(协议/网络地址/端口)只允许使用一次。 (os error 10048)
```

**解决方案**：
- 检查端口 8080 是否被占用
- 使用 `netstat -ano | findstr :8080` 查找占用进程
- 关闭占用进程或修改后端配置中的端口

### 问题 2：前端显示"未连接服务器"

**可能原因**：
- 后端服务未启动
- 后端服务地址配置错误
- 防火墙阻止连接

**解决方案**：
1. 确认后端服务正在运行
2. 检查前端 API 服务地址配置
3. 检查防火墙设置

### 问题 3：无法加载模型列表

**可能原因**：
- Ollama 服务未启动
- Ollama 中没有下载模型

**解决方案**：
1. 确认 Ollama 服务正在运行
2. 使用 `ollama list` 查看可用模型
3. 使用 `ollama pull <model-name>` 下载模型

### 问题 4：聊天响应很慢

**可能原因**：
- 模型文件较大
- 计算资源不足

**解决方案**：
1. 使用较小的模型（如 qwen3.5:0.8b）
2. 关闭其他占用资源的程序
3. 考虑使用 GPU 加速

### 问题 5：LaTeX 公式不显示

**可能原因**：
- flutter_tex 插件未正确安装
- 数学公式格式错误

**解决方案**：
1. 运行 `flutter pub get` 确保依赖安装
2. 检查 LaTeX 公式格式是否正确
3. 使用标准的 LaTeX 语法

cargo install cargo-watch
